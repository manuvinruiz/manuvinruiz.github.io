Adding an action

By default, a PDDL action in ROSPlan is dispatched as an ActionDispatch message.

Topic Name: kcl_rosplan/action_dispatch
Message Type: rosplan_dispatch_msgs/ActionDispatch

Similarly, feedback is published as an ActionFeedback message.

Topic Name: kcl_rosplan/action_feedback
Message Type: rosplan_dispatch_msgs/ActionFeedback

To link a new action, there is one of two ways. Either:

  1. Write a new node that uses these two topics; or
  2. Write a subclass of rosplan_action_interface::ActionInterface

Option 2 (writing a subclass) is the simplest method. The class rosplan_action_interface::ActionInterface handles recieving the action, action feedback, and updating the knowledge base. Linking an action to ROSPlan in this way is described in detail in the ROSPlan github wiki
